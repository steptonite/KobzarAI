## 1. Три моделі під твій Mac (різні тіри, в межах 8ГБ)

Реальна стеля на 8ГБ = **4B Q4_K_M**. Більшого немає сенсу — RAM, не диск, є межею.
Три ролі:

| Тір | Модель | Швидк. | Роль |
|---|---|---|---|
| **Нано** | `gemma3:1b` | **61 tok/s** | блискавичні чернетки, заголовки, перефраз, прості пояснення |
| **Щоденна (укр)** | `MamayLM-Gemma-3-4B` (INSAIT) | 25 tok/s | **дефолт на все українське** — уроки, тексти, креатив, стислі задачі |
| **Універсал** | `gemma3:4b` | 26 tok/s | код, точний формат, RU, термінологія, міркування |

**Чому (з фактів бенчу):**
- **gemma3:1b** — найшвидша. АЛЕ завалила імовірність (0.25 замість 5/14), зіпсувала JSON
  (`{"Python":1991}` без потрібних ключів), tuple назвала «туп/тупила», грам. зриви («напої»).
  → лише для простого/швидкого, НЕ для математики/точності/термінів.
- **MamayLM-4B** — найжива українська, найкращі стислі слогани, чіткий формат «рівно 3», вірна
  логіка ймовірності (5/8·4/7). RU трохи слабша за gemma3:4b.
- **gemma3:4b** — найсильніший універсал: єдина написала `O(log n)` явно, найкраща термінологія
  (`кортеж`, mutability), ідеальний формат, грамотна російська, вірна комбінаторика C(8,2)=28.

**Що прибрати/тримати:**
- `qwen3:4b` → **прибрати з робочого набору** (причина нижче, розділ 2).
- `qwen3-abliterated:4b` → лишити лише як uncensored-запас. Та сама біда з думанням.
- Краща uncensored-опція = **`mlabonne/gemma-3-abliterated`** (gemma-база = відповідає одразу
  укр, без think-оверхеду). Не бенчена, але архітектурно правильніша за qwen-abliterated.

## 2. Чому qwen3 випав: думання НЕ вимикається на Ollama 0.9.3

qwen3 — reasoning-модель: перед відповіддю «думає» вголос **англійською/російською** і на ліміті
токенів витрачає весь бюджет на роздуми, не доходячи до відповіді. У бенчу майже всюди — суцільний
англ-монолог, української відповіді немає.

**Спробували вимкнути думання — не вийшло (емпірично, цю сесію):**
- `"think": false` у `/api/generate` → **ігнор** (підтримка цього параметра додана пізніше, його
  білд застарий). Відповідь усе одно з `<think>`, рос. текстом, `done_reason:"length"`.
- `/no_think` суфікс у промпті через `/api/generate` → **ігнор**, думає англійською, ще й пише
  «user said no_think so I shouldn't add extra thought» — і все одно думає.
- `/no_think` через `/api/chat` (де діє chat-шаблон qwen) → **ігнор**, те саме.

**Висновок:** на поточному Ollama 0.9.3 думання qwen3 заглушити нічим. Два шляхи, якщо qwen
колись потрібен:
1. **Оновити Ollama** до білда, де `think:false` працює (Messages API + think-toggle — це версії
   ~січня 2026+). Це окреме рішення (homebrew upgrade) — робити лише за явної згоди.
2. **Просто не використовувати qwen для коротких задач** — gemma-família думання не має взагалі,
   відповідає одразу. Для тебе це й є відповідь: **бери gemma-стек**.

## 3. Cherry «Agent mode» локально — остаточно

**Локальний агент у Cherry на твоєму Mac — ні, архітектурно.** Не баг налаштувань:
- Cherry агент-режим працює лише з провайдером типу **Anthropic** або **CherryIN**. OpenAI-сумісні
  (а Ollama саме такий) **не зʼявляються** у пікері моделей агента.
- Під капотом агент шелл-аутить у Claude Code CLI / говорить Anthropic Messages API
  (`/v1/messages`). Твій Ollama 0.9.3 на цей ендпойнт віддає **404** (підтримку додано ~січ-2026).
- Навіть якби підключив — 4B-моделі **ненадійні в tool-calling** (рвуть JSON інструментів).
  Це не про залізо: баг ollama#15390 показує злам gemma на 2×RTX 3060 (24ГБ VRAM). Питання
  спроможності моделі, не RAM.

**Як працювати з агентами — варіанти:**

| Шлях | Локально? | Що дає | Кому |
|---|---|---|---|
| **CherryIN** (вбудований шлюз Cherry) | ні (хмара) | 1 акаунт → Claude/Gemini/GPT/DeepSeek, open-source до −40%, 500k безкошт. на старт | **рекомендую для агентів** — найменше тертя, агент-режим Cherry оживає одразу |
| Anthropic API-ключ у Cherry | ні (хмара) | прямий Claude в агенті | якщо вже є ключ |
| **Ollama → лише Chat** | так | локальний чат gemma/MamayLM | офлайн-робота, тексти, код-помічник без агентності |

**Якщо хочеш САМЕ локальний агент потестити** (попри слабкість 4B) — Cherry для цього не підходить,
треба інший раннер з підтримкою Ollama tool-calling:
- **AnythingLLM** (agent skills, локально, Ollama-бекенд) — найпростіше поставити й глянути.
- **Open WebUI** (tools/functions) або **Goose** (CLI-агент) — теж їдять Ollama.
- Чесне очікування: на 8ГБ + 4B багатокрокові інструменти **будуть спотикатись**. Тест — щоб
  побачити стелю на власні очі, не для продакшну.

**Підсумок:** агенти — через **CherryIN/хмару**; Ollama тримай для офлайн-**Chat**. Локальний
агент = радше експеримент в AnythingLLM, ніж робочий інструмент на цьому залізі.

## 4. Як перезапустити тест (коли захочеш)

```bash
bash ~/.ollama/start-ollama.sh                     # підняти Ollama (ручний старт)
python3 bench/bench.py                              # повний прогін, 5 моделей
BENCH_MODELS="gemma3:4b,gemma3:1b" python3 bench/bench.py   # лише обрані
BENCH_NOTHINK=1 python3 bench/bench.py              # спроба без думання (на 0.9.3 не діє — див. розд.2)
BENCH_NUM_PREDICT=700 python3 bench/bench.py        # довша стеля відповіді
```
Звіти лягають у `bench/results/run_<stamp>.md` (+ .json). `results/` у .gitignore.

## 5. Догон 2026-06-18 23:5x — нові моделі, VL, no_think, аблітерація

### no_think — РОЗВ'ЯЗАНО без хаків
- `think:false`/`/no_think` на Ollama 0.9.3 — мертві (issues ollama#14617, #14809 досі open).
  Причина чому **на айфоні працює**: PocketPal/Enclave = llama.cpp напряму, чесний jinja-шаблон.
  Ollama = власний Go-шаблон, баг.
- **Рішення = `qwen3:4b-instruct-2507-q4_K_M`** (стягнуто, 2.5G). `<think>` НЕ існує by design
  (`enable_thinking` не потрібен). Перевірено: HAS_THINK=False, 27 tok/s, чиста укр. **Заміняє
  старий `qwen3:4b` начисто.** (є й `…-thinking-2507` — це навпаки, не брати.)
- Альтернатива якщо колись треба сам qwen-перемикач: upgrade Ollama (homebrew, окрема згода).

### VL (зір) — що реально вміє на 0.9.3
| Модель | Зір? | Нота |
|---|---|---|
| `gemma3:4b` | ✓ працює | але 4B-зір слабкий: зациклюється на тексті, не дочитує |
| `gemma3n:e4b` | ✓ працює | галюцинує — вигадав «трикутник», прочитав промпт замість картинки |
| `mlabonne/gemma-3-4b-abliterated` | ✗ | GGUF без mmproj → `missing data for image input`, тільки текст |
| `MamayLM-4B` | ✗ | те саме, текст-онлі GGUF |
| `huihui qwen3-vl-abliterated:4b` | ✗✗ | **взагалі не вантажиться** (`unable to load model`) — арх qwen3-vl новіша за Ollama 0.9.3 |

**VL-вердикт:** надійного локального зору на 8ГБ+0.9.3 НЕМА. gemma3:4b/gemma3n technically бачать,
але на 4B галюцинують і ламаються. Справжній VL (qwen3-vl) вимагає **upgrade Ollama**. Поки зір
лишається слабким місцем — для серйозного OCR/опису краще хмара.

### gemma3n:e4b — вердикт
Укр відмінна («пиріг»-аналогія), але **18 tok/s vs 26 у gemma3:4b**, диск 7.5G, RAM важчий.
Не б'є MamayLM по укр і повільніший за gemma3:4b. **Тримати не варто** — кандидат на видалення.

### Аблітерація — РОБОЧА vs ЗЛАМАНА
Тест: темна проза, indus-лірика з матом, чорний гумор, гострий погляд, фішинг-захист (5 промптів).
- **gemma3:4b (цензурована база) — виконала ВСІ 5 без відмов**, найкраща зв'язність укр. Тобто для
  легітимного edgy-креативу + захисної безпеки аблітерація часто й не потрібна.
- **`mlabonne/gemma-3-4b-abliterated` — РОБОЧА якісна**: усе виконала, без think-оверхеду, добра укр.
  Дрібні артефакти зв'язності («їхньомиленими», «точкутисну») — ціна аблітерації. **Це твій
  uncensored-пік.**
- **`huihui qwen3-abliterated:4b` — аблітерація працює (0 відмов), АЛЕ зламана думанням**: весь
  бюджет токенів іде в англ `<think>`, укр-відповідь не дописується. На 0.9.3 — непридатна.
- **`huihui qwen3-vl-abliterated:4b` — не вантажиться (500).**

**Рекомендація по набору (uncensored):** тримати `mlabonne/gemma-3-4b-abliterated`. Викинути обидві
qwen-аблітерації (одна зламана думанням, друга не вантажиться).

### Підсумкова пропозиція по чистці (потрібна твоя згода на видалення)
ВИКИНУТИ: `qwen3:4b` (→ instruct-2507), `huihui qwen3-abliterated:4b` (думання),
`huihui qwen3-vl-abliterated:4b` (не вантажиться), `gemma3n:e4b` (повільний, без переваги).
ТРИМАТИ: `gemma3:4b`(універсал+єдиний робочий зір), `MamayLM-4B`(укр), `gemma3:1b`(нано),
`qwen3:4b-instruct-2507`(швидкий без-думання), `mlabonne-abliterated`(uncensored), `nomic-embed`.
